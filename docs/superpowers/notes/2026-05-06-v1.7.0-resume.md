## Lessons from the session

**What works:** Direct in-session editing using `Edit` and small `awk`/`sed` cuts. Pattern X cuts come together fast when you read the SKILL.md once, identify cut targets up front, then edit each section.

**What doesn't work:** Long-prompt subagent dispatches for restructures keep hitting `OverloadedError` from the API. After two failures (Tasks 5 and 6) we switched to direct work and it's been smoother.

**Pattern X subtleties learned:**
- The validator's orphan check uses regex `references[\\/]<file>.md`. Links must be relative with forward slashes.
- Replacing a section is a 2-step pattern: write the reference file with header `# Title\n\nReference for skills/<name>/SKILL.md — <scope>.\n\n> ← Back to [SKILL.md](../SKILL.md)\n\n---\n\n<content>` THEN edit SKILL.md to replace the section body with a 2-3 sentence summary + link.
- `awk NR==<line>` cuts break across multiple sections in one pass because line numbers shift after the first cut. Do them one at a time, or use grep to refind lines after each cut.
- Trim aggressively: if SKILL.md is at 16.5 KB after the planned cuts, trim secondary tables and prose paragraphs (the validator threshold is 16 KB / 16384 bytes — strict `>=`).
