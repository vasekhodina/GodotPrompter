## Lessons (carry-over from v1.7.0/v1.7.1, still apply)

- Direct `Edit` + `sed` works better than subagents for restructures (subagents kept overloading on this scale of work)
- Always reverse-order the `sed -i 'A,Bd'` cuts so line numbers stay valid
- After each restructure: count-tokens grep, validator grep, then commit
- Section moves don't break `csharp-parity-missing` warnings if both languages travel together to references
- The validator only checks SKILL.md sections — moved-to-references parity warnings disappear silently (this is fine, by design)
