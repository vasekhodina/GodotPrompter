# Token Budget — GodotPrompter Skills and Agents

This page documents the token cost of every skill and agent in the repository, so users can plan their context budget when working with GodotPrompter.

## What this is

GodotPrompter ships **44 skills** and **8 agents** as Markdown files (45 skills and 9 agents from v1.7.0 onward). When an AI coding assistant (Claude Code, Copilot, Gemini, Cursor, etc.) loads a skill, every byte of the skill's `SKILL.md` enters the model's context. Loading 3-4 skills is normal for a single feature task — at typical sizes that's 20-30k tokens before the user prompt even starts.

To keep skills loadable in combination, v1.7.0 introduced a **16 KB budget** for `SKILL.md` files. Skills that exceed the budget are progressively restructured using the **Pattern X** approach — core SKILL.md (canonical recipe + distinguishing choices + anti-patterns) plus `references/<topic>.md` files that load only when an agent explicitly opens them.

The validator (`node scripts/validate-skills.mjs`) emits a `[token-budget-exceeded]` warning for any over-budget SKILL.md. Reference files are unrestricted in size — they're only loaded on demand.

## How to read this

A typical agent invocation reads:

1. The agent's own `agents/<name>.md` (1 file, usually under 6 KB / ~1.4k Claude tokens)
2. The agent's primary skill's `SKILL.md` (1 file)
3. 1-3 cross-referenced skills' `SKILL.md` files (because agents are taught to read related skills before designing or coding)

At p50 across the catalog, that's roughly **20-25k Claude tokens** loaded before the conversation begins. At p90 (heavy agents like `godot-game-dev` that touch many domains), it's **35-45k**. The 16 KB / ~4k-token budget on a single SKILL.md keeps a 4-skill load comfortably under 20k tokens after restructure.

## Reducing your token use

If you only need part of a heavy skill:

- Read just the relevant `references/<topic>.md` rather than the full SKILL.md (only available for restructured skills — see `docs/superpowers/notes/2026-05-06-token-budget-debt.md` for which skills have references and which are still pending)
- Ask the agent to skip cross-references it doesn't need for the immediate task
- For agent-driven flows: tell the agent up front which skill section you care about
